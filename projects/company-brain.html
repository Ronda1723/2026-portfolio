Company Brain

A per-employee brain, native to Slack.

Not another chatbot. A memory layer that lives where work already happens — Slack threads, meeting transcripts, internal docs — and knows which parts belong to which employee. Six wedge use cases picked carefully. MCP day one. Voice left out of v1 on purpose.

The problem

Every team has the same problem: knowledge lives in people, not in systems. Onboarding is asking around. Decisions get re-litigated because no one remembers the last conversation. Search is broken because it's per-tool, not per-context.

The temptation is to build "one chatbot to rule them all." That's the wrong shape. People don't want a chatbot — they want a brain that remembers for them.

The approach

Six wedge use cases picked carefully — not "answers all questions," but six specific moments where memory beats search:

  1. "What did we decide about X?" — surface the last canonical decision thread.
  2. Meeting recall — what was said, by whom, with action items extracted.
  3. Onboarding shortcut — a personalized FAQ generated from past threads.
  4. Context handoff — when picking up someone else's work, get a focused brief.
  5. Standup summarizer — what changed since last time, per project.
  6. Personal log — a private feed of your own contributions for review/retrospective.

Voice is intentionally not in v1. A brain has to feel trustworthy first; voice raises the trust bar and adds latency we don't need to debug yet.

Architecture, briefly

Slack-first ingestion (threads, channels, DMs) with per-employee scope enforcement at the storage layer. MCP day one means any compliant agent (Claude, GPT, local) can read the brain as a tool. A local agent runs in P1 — for privacy-sensitive teams who want the embedding/retrieval layer on their hardware.

Where it is now

Vision v4 is locked. We're heads-down on the Slack ingestion pipeline and the per-employee scoping model. Early access opens to a small cohort of design partners first.
