Uxie

Voice-first writing for Mac & Windows.

Press a key, speak, and clean text appears anywhere you type. A streaming speech-to-text app with a key cache, interim transcripts, and dispatch-on-release — built so the magic happens under 300ms.

The problem

Most voice-to-text tools feel like dictation — slow, robotic, broken by punctuation. You speak, you wait, you fix. That's not writing — that's transcription cleanup.

I wanted a tool that felt like an extension of typing, not a replacement for it. Press a key, talk for a sentence, release. Text should already be there by the time your finger lifts.

The approach

Three obsessions: latency, parity, and polish.

Latency. Streaming Deepgram Nova-3 with interim transcripts. The model returns guesses as you speak — we cache the last reliable cut and dispatch on key release. The result: text appears as you let go of the key, not after a round-trip.

Parity. Same version, same day, Mac and Windows. Built on Tauri so the binary's small and the signing pipeline is automatable. No "Mac-first, Windows when we get to it."

Polish. Signed, notarized, no scary unsigned-app dialogs. Quiet menubar app. Hotkey customization that doesn't fight the OS.

What shipped

  • Streaming STT pipeline with sub-300ms perceived latency
  • Mac DMG (signed + notarized) and Windows EXE (signed), released as one version
  • Key cache so dispatch-on-release feels instant
  • Per-app hotkey scoping (different shortcut in Slack vs in your editor)
  • Backend on Railway, all keys server-side — never bundled with the app

What I'm working on next

Word-level edit gestures (say "scratch that"), per-language model swapping, and a small offline fallback for short utterances. The goal is to keep raising the floor without losing the "press, speak, done" rhythm.
